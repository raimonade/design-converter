## 4. Connection Modes

FigmaWriter supports three connection modes for executing Plugin API JavaScript in Figma.
Each mode has different trade-offs between automation, setup complexity, and use cases.

| Mode | Transport | Port | Automation | Use Case |
|------|-----------|------|------------|----------|
| **script** | File I/O | — | Manual paste | One-off conversions, debugging, handoff |
| **bridge** | WebSocket (RFC 6455) | 9224 | Full | Automated pipelines, direct Python → Figma |
| **http** | HTTP POST | 9223 | Full | MCP tools, symmetric with Paper/Pencil |

### 4.1 Script Mode

**Default mode.** Generates a self-contained JavaScript IIFE (Immediately Invoked Function Expression)
saved to disk. The user manually pastes the script into Figma's Plugin Console.

#### When to Use

- One-off design conversions
- Debugging UNNode → Figma mapping
- Design handoff where developers inspect generated code
- Environments without network access to Figma

#### How It Works

```
UNNode tree → _FigmaCodeEmitter → .js IIFE file → User pastes → Figma Console
```

1. `FigmaWriter.write_node()` receives a UNNode tree
2. `_FigmaCodeEmitter` walks the tree and generates JavaScript
3. Script is saved to `output_dir` (defaults to CWD)
4. User opens Figma → Plugins → Development → Open Console (Cmd+Opt+I)
5. User pastes script and presses Enter

#### Output File Format

```js
// Generated by FigmaWriter — paste into Figma ▶ Plugins ▶ Console
// Node : 'Button'
// Type : FRAME
// Date : 2026-03-03 14:32:00
//
// Steps:
//   1. Open Figma
//   2. Plugins ▶ Development ▶ Open Console  (Cmd+Opt+I)
//   3. Paste this script and press Enter
//
(async () => { try {
  await figma.loadFontAsync({family:"Inter",style:"Regular"});
  const _par = figma.currentPage;
  const n0 = figma.createFrame();
  n0.name = "Button";
  n0.resize(120, 40);
  n0.layoutMode = "HORIZONTAL";
  n0.itemSpacing = 8;
  n0.fills = [{type:'SOLID',color:{r:0.2,g:0.4,b:0.8,a:1}}];
  _par.appendChild(n0);
  return {success:true,nodeId:n0.id,nodeName:n0.name};
} catch(e) { return {success:false,error:String(e)}; }
})();
```

#### CLI Usage

```bash
# Default mode is script
python3 converter.py paper:TO-0 figma:

# Explicit mode
python3 converter.py paper:TO-0 figma: --figma-writer-mode=script

# Custom output directory
python3 converter.py paper:TO-0 figma: --output-dir=./output
```

#### Python API

```python
from converter import DesignConverter

conv = DesignConverter(figma_writer_mode="script", figma_output_dir="./output")
result = conv.convert("paper:TO-0", "figma:")
print(result.output)  # → "./output/Button.js"
```

### 4.2 Bridge Mode

**Direct WebSocket mode.** FigmaWriter starts a minimal RFC 6455 WebSocket server.
The Desktop Bridge plugin (running inside Figma) connects as a client.

#### When to Use

- Automated CI/CD pipelines
- Real-time design sync
- Scripts that need immediate feedback
- When you control both Python and Figma environments

#### How It Works

```
┌─────────────────┐                    ┌─────────────────────┐
│  FigmaWriter    │  ws://127.0.0.1:9224  │  Desktop Bridge   │
│  (Python WS     │◄────────────────────►│  Plugin (Figma)   │
│   SERVER)       │  RFC 6455 WebSocket   │  (WS CLIENT)      │
└─────────────────┘                    └─────────────────────┘
```

1. `FigmaWriter.connect()` starts WebSocket server on port 9224
2. Waits up to `connect_timeout` (default 60s) for plugin to connect
3. Plugin connects → handshake completes
4. `write_node()` sends `EXECUTE_CODE` request
5. Plugin executes JS in Figma sandbox, returns result
6. Returns created node ID

#### WebSocket Protocol

Pure Python stdlib implementation — no external dependencies (`websockets`, `aiohttp`, etc.).

**Request format:**
```json
{
  "id": "py_1_1709487600000",
  "method": "EXECUTE_CODE",
  "params": {
    "code": "(async () => { ... })()",
    "timeout": 30000
  }
}
```

**Success response:**
```json
{
  "id": "py_1_1709487600000",
  "result": {
    "success": true,
    "nodeId": "12:45",
    "nodeName": "Button"
  }
}
```

**Error response:**
```json
{
  "id": "py_1_1709487600000",
  "error": "Cannot read property 'resize' of null"
}
```

#### CLI Usage

```bash
# Start bridge mode (requires Desktop Bridge plugin running)
python3 converter.py paper:TO-0 figma: --figma-writer-mode=bridge

# Custom port
python3 converter.py paper:TO-0 figma: --figma-writer-mode=bridge --bridge-port=9225

# Custom timeouts
python3 converter.py paper:TO-0 figma: --figma-writer-mode=bridge \
    --bridge-port=9224 \
    --connect-timeout=120
```

#### Python API

```python
from converter import DesignConverter

conv = DesignConverter(
    figma_writer_mode="bridge",
    figma_bridge_port=9224,
    figma_connect_timeout=60.0,
    figma_bridge_timeout=30.0,
)

# Context manager ensures disconnect even on error
with conv:
    result = conv.convert("paper:TO-0", "figma:")
    print(result.output)  # → "12:45" (Figma node ID)
```

#### Setup Requirements

1. Install Desktop Bridge plugin in Figma
2. Open Figma and run the plugin
3. Configure plugin to connect to `ws://127.0.0.1:9224`
4. Run Python converter — server starts and waits for connection

### 4.3 HTTP Mode

**HTTP bridge mode.** Uses an external HTTP → WebSocket proxy server (`figma-bridge-server`).
This mode is symmetric with Paper (HTTP on :29979) and enables any HTTP client to write to Figma.

#### When to Use

- MCP tools (figma-console MCP uses this mode)
- Multiple clients writing to Figma
- When you want HTTP semantics instead of raw WebSocket
- Integration with existing HTTP infrastructure

#### How It Works

```
┌─────────────────┐   HTTP POST    ┌─────────────────┐   WebSocket   ┌─────────────────┐
│  FigmaWriter    │───────────────►│  bridge_server  │◄─────────────►│  Desktop Bridge │
│  (HTTP client)  │   :9223        │  (HTTP → WS)    │    :9224      │  Plugin         │
└─────────────────┘                └─────────────────┘               └─────────────────┘
```

1. `figma-bridge-server` runs on port 9223 (HTTP) and 9224 (WebSocket)
2. Desktop Bridge plugin connects to WebSocket side
3. `FigmaWriter.connect()` verifies server is running and plugin is connected
4. `write_node()` POSTs to `/execute` endpoint
5. Bridge server forwards to plugin via WebSocket
6. Returns created node ID

#### HTTP API

**Health check:**
```bash
curl http://127.0.0.1:9223/health
# → {"status":"ok","plugin_connected":true}
```

**Execute code:**
```bash
curl -X POST http://127.0.0.1:9223/execute \
  -H "Content-Type: application/json" \
  -d '{"code":"figma.createRectangle()","timeout":30000}'
# → {"success":true,"result":{"nodeId":"15:2"}}
```

**Status endpoint:**
```bash
curl http://127.0.0.1:9223/status
# → {"status":"ok","plugin_connected":true,"uptime_seconds":3600}
```

#### CLI Usage

```bash
# Start HTTP bridge server first (separate terminal)
python3 adapters/figma/bridge_server.py --port 9223

# Run converter in HTTP mode
python3 converter.py paper:TO-0 figma: --figma-writer-mode=http

# Custom HTTP port
python3 converter.py paper:TO-0 figma: \
    --figma-writer-mode=http \
    --http-bridge-port=9223
```

#### Python API

```python
from converter import DesignConverter

conv = DesignConverter(
    figma_writer_mode="http",
    figma_http_bridge_port=9223,
)

result = conv.convert("paper:TO-0", "figma:")
print(result.output)  # → "12:45" (Figma node ID)
```

#### Direct HTTP Client Usage

```python
from adapters.figma.http_bridge import FigmaBridgeClient

client = FigmaBridgeClient(port=9223)

# Check connection
if client.is_connected():
    result = client.execute_code("figma.createRectangle()")
    print(result.node_id)  # → "15:2"

# Convenience methods
client.create_frame("MyFrame", x=0, y=0, width=200, height=100)
client.create_text("Hello", x=10, y=10, font_size=16)
```

### 4.4 Mode Comparison

| Feature | script | bridge | http |
|---------|:------:|:------:|:----:|
| Requires plugin running | No | Yes | Yes |
| Requires server process | No | No | Yes |
| Automated execution | No | Yes | Yes |
| Returns node ID | No | Yes | Yes |
| Multiple clients | N/A | No | Yes |
| Network accessible | No | Local only | Configurable |
| MCP compatible | No | No | Yes |
| Error handling | Manual | Auto | Auto |
| Setup complexity | Low | Medium | Medium |

### 4.5 Timeout Configuration

| Parameter | Default | Mode | Description |
|-----------|---------|------|-------------|
| `connect_timeout` | 60.0s | bridge | Time to wait for plugin to connect |
| `bridge_timeout` | 30.0s | bridge, http | Time to wait for code execution response |
| HTTP client timeout | 30.0s | http | HTTP request timeout |

```python
conv = DesignConverter(
    figma_writer_mode="bridge",
    figma_connect_timeout=120.0,  # Wait 2 min for plugin
    figma_bridge_timeout=60.0,    # Wait 1 min for execution
)
```

## 5. Workflow Examples

This section provides copy/paste-ready examples for common conversion workflows.
Each example shows both CLI and Python API approaches.

### 5.1 Paper → Figma (Script Mode)

The simplest workflow — generate a JavaScript file and paste it into Figma.

#### CLI

```bash
# Convert Paper component to Figma script
python3 services/design-converter/converter.py paper:TO-0 figma:

# With custom output location
python3 services/design-converter/converter.py paper:TO-0 figma: \
    --output-dir=./figma-scripts

# Dry run to inspect the UNNode tree first
python3 services/design-converter/converter.py paper:TO-0 figma: --dry-run
```

#### Python

```python
from converter import DesignConverter

# Create converter in script mode (default)
converter = DesignConverter(
    figma_writer_mode="script",
    figma_output_dir="./figma-scripts",
)

# Perform conversion
result = converter.convert("paper:TO-0", "figma:")

if result.success:
    print(f"Script saved to: {result.output}")
    # → "./figma-scripts/TO-0.js"
else:
    print(f"Conversion failed: {result.error}")
```

### 5.2 Paper → Figma (Bridge Mode)

Automated workflow with direct Python → Figma connection via WebSocket.

#### CLI

```bash
# Ensure Desktop Bridge plugin is running in Figma first
python3 services/design-converter/converter.py paper:TO-0 figma: \
    --figma-writer-mode=bridge \
    --bridge-port=9224

# With extended timeouts for large designs
python3 services/design-converter/converter.py paper:TO-0 figma: \
    --figma-writer-mode=bridge \
    --connect-timeout=120 \
    --bridge-timeout=60
```

#### Python

```python
from converter import DesignConverter

# Create converter in bridge mode
converter = DesignConverter(
    figma_writer_mode="bridge",
    figma_bridge_port=9224,
    figma_connect_timeout=60.0,
    figma_bridge_timeout=30.0,
)

# Use context manager for automatic cleanup
with converter:
    result = converter.convert("paper:TO-0", "figma:")

    if result.success:
        print(f"Created Figma node: {result.output}")
        # → "12:45" (Figma node ID)
    else:
        print(f"Failed: {result.error}")
```

### 5.3 Paper → Figma (HTTP Mode)

HTTP-based workflow for MCP tools and multi-client scenarios.

#### CLI

```bash
# Start bridge server (terminal 1)
python3 services/design-converter/adapters/figma/bridge_server.py --port 9223

# Run conversion (terminal 2)
python3 services/design-converter/converter.py paper:TO-0 figma: \
    --figma-writer-mode=http \
    --http-bridge-port=9223
```

#### Python

```python
from converter import DesignConverter

# Create converter in HTTP mode
converter = DesignConverter(
    figma_writer_mode="http",
    figma_http_bridge_port=9223,
)

# No context manager needed — HTTP is stateless
result = converter.convert("paper:TO-0", "figma:")

if result.success:
    print(f"Created node ID: {result.output}")
```

### 5.4 Figma → Paper

Read from Figma via REST API and write to Paper via MCP.

#### CLI

```bash
# Convert Figma node to Paper component
python3 services/design-converter/converter.py figma:abc123:45:67 paper:

# With verbose output
python3 services/design-converter/converter.py figma:abc123:45:67 paper: --verbose
```

#### Python

```python
from converter import DesignConverter
from adapters.figma import FigmaReader
from adapters.paper import PaperWriter

# Option 1: High-level converter API
converter = DesignConverter()
result = converter.convert("figma:abc123:45:67", "paper:")

# Option 2: Low-level reader/writer API
reader = FigmaReader(access_token="your-figma-token")
writer = PaperWriter(mcp_port=29979)

# Read UNNode tree from Figma
tree = reader.read_node(file_key="abc123", node_id="45:67")

# Write to Paper
writer.write_node(tree, output_path="NewComponent")
```

### 5.5 Batch Conversion

Convert multiple components in a single run.

#### CLI

```bash
# Convert all components in a Paper file
python3 services/design-converter/converter.py paper:DesignFile:* figma: \
    --figma-writer-mode=script \
    --output-dir=./batch-output

# Use a spec file for batch operations
python3 services/design-converter/converter.py --spec-file=./conversions.json
```

#### Python

```python
from converter import DesignConverter, ConvertSpec

# Define batch conversions
specs = [
    ConvertSpec(source="paper:TO-0", target="figma:"),
    ConvertSpec(source="paper:TO-1", target="figma:"),
    ConvertSpec(source="paper:TO-2", target="figma:"),
]

converter = DesignConverter(figma_writer_mode="script")

results = []
for spec in specs:
    result = converter.convert(spec.source, spec.target)
    results.append(result)
    print(f"{spec.source} → {result.success}")

# Summary
successful = sum(1 for r in results if r.success)
print(f"Converted {successful}/{len(results)} components")
```

### 5.6 Custom UNNode Pipeline

Build custom pipelines by manipulating the UNNode tree directly.

#### Python

```python
from ir.nodes import UNNode, NodeType, UNColor, UNSolidFill, UNPadding
from adapters.figma import FigmaWriter
from adapters.paper import PaperReader

# Read from Paper
reader = PaperReader(mcp_port=29979)
tree = reader.read_node(file_key="DesignFile", node_id="Button")

# Modify the UNNode tree programmatically
for node in tree.children:
    if node.type == NodeType.FRAME:
        # Add padding
        node.padding = UNPadding(top=16, right=24, bottom=16, left=24)

        # Change fill color
        if node.fills:
            node.fills[0] = UNSolidFill(
                color=UNColor(r=0.2, g=0.4, b=0.8, a=1.0)
            )

# Write modified tree to Figma
writer = FigmaWriter(mode="bridge", bridge_port=9224)
with writer:
    node_id = writer.write_node(tree)
    print(f"Created: {node_id}")
```

## 7. Known Issues

This section documents known bugs, limitations, and workarounds in the design-converter ecosystem.

### 7.1 HTTP Bridge Timeout Bug (BRIDGE-001)

**Status:** 🔴 Open · **Severity:** High · **Affected Mode:** HTTP (bridge_server.py)

#### Description

The HTTP bridge server (`bridge_server.py`) may fail to return responses within the expected timeout window when executing long-running Figma operations. This manifests as `Gateway Timeout (504)` errors even when the Figma plugin successfully completes the operation.

#### Root Cause

The `_send_to_plugin()` method in `bridge_server.py` adds a fixed 5-second buffer to the client-requested timeout:

```python
# Line 492 in bridge_server.py
result = await asyncio.wait_for(future, timeout=timeout_ms / 1000 + 5)
```

This buffer may be insufficient for:
1. Complex node trees requiring multiple font loads
2. Operations on large canvases with many children
3. Network latency between the bridge server and Figma plugin

#### Symptoms

- HTTP 504 Gateway Timeout responses from `/execute` endpoint
- Error message: `"No response from Figma within {timeout_ms}ms"`
- Operations that succeed in Figma but fail to return results to the client

#### Workaround

Increase the `timeout_ms` parameter when calling the `/execute` endpoint:

```bash
# Instead of default 30000ms, use 60000ms for complex operations
curl -X POST http://localhost:9223/execute \
  -H "Content-Type: application/json" \
  -d '{"code": "...", "timeout": 60000}'
```

Or use the retry mechanism (enabled by default):

```bash
# Retry is enabled by default, but can be explicitly set
curl -X POST http://localhost:9223/execute \
  -H "Content-Type: application/json" \
  -d '{"code": "...", "timeout": 30000, "retry": true}'
```

#### Retry Behavior

The `_send_to_plugin_with_retry()` method provides automatic retry with exponential backoff:
- **Attempt 1:** Immediate
- **Attempt 2:** After 1.5 seconds
- **Attempt 3:** After 3.0 seconds (cumulative: 4.5s)
- **Max attempts:** 3 (configurable via `MAX_RETRIES`)

#### Fix Planned

A future update will:
1. Make the timeout buffer configurable via CLI flag
2. Implement adaptive timeout based on operation complexity
3. Add early heartbeat responses for long-running operations

## 8. UNNode IR Reference

The Universal Node Tree (UNT) is the intermediate representation (IR) that sits at the core of the design-converter architecture. Every design tool (Figma, Paper, Pencil) maps to and from this unified tree structure.

### 8.1 Philosophy

The IR follows these principles:

- **Everything is a node** — Modeled after Figma's SceneNode hierarchy
- **Fills/strokes/effects are first-class lists** — Ordered bottom-to-top, like Figma
- **Layout is flex-inspired** — Matches auto-layout semantics across all three tools
- **Variables/tokens tracked as bindings** — Preserves round-trip fidelity
- **No tool-specific concepts** — The IR is tool-agnostic

### 8.2 Core Node Structure

The `UNNode` dataclass is the primary building block:

### 8.4 Helper Dataclasses

#### UNColor

Normalized RGBA color with 0.0–1.0 float channels.

```python
from ir.nodes import UNColor

# From hex
color = UNColor.from_hex("#FF5733")
color = UNColor.from_hex("#FF573380")  # With alpha

# Direct construction
color = UNColor(r=1.0, g=0.34, b=0.2, a=1.0)

# Conversion methods
color.to_hex()           # → "#FF5733"
color.to_css_rgba()      # → "rgba(255,87,51,1)"
color.to_figma_rgb()     # → {"r": 1.0, "g": 0.34, "b": 0.2}
```

#### UNSize

Width/height with sizing mode (fixed, hug, fill).

```python
from ir.nodes import UNSize, SizingMode

# Factory methods
size = UNSize.fixed(200)    # Fixed 200px
size = UNSize.hug()         # Fit content
size = UNSize.fill()        # Fill parent

# Conversion
size.to_css()       # → "200px" / "fit-content" / "100%"
size.to_figma()     # → {"primaryAxisSizingMode": "AUTO"} etc.
size.to_pencil()    # → 200 / "fit_content" / "fill_container"
```

#### UNPadding

Per-side padding values.

```python
from ir.nodes import UNPadding

# Factory methods
padding = UNPadding.all(16)              # All sides 16px
padding = UNPadding.xy(24, 16)           # H: 24px, V: 16px
padding = UNPadding.sides(top=8, left=12)  # Individual sides

# Conversion
padding.to_css()      # → "16px" / "16px 24px" / "8px 12px 8px 12px"
padding.is_zero()     # → True if all sides are 0
```

#### UNCornerRadius

Per-corner radius values.

```python
from ir.nodes import UNCornerRadius

# Factory methods
radius = UNCornerRadius.all(8)           # All corners 8px
radius = UNCornerRadius.sides(tl=4, br=12)  # Individual corners

# Methods
radius.is_uniform()   # → True if all corners equal
radius.value()        # → Returns uniform value (raises if not uniform)
radius.to_css()       # → "8px" / "4px 4px 12px 12px"
```
